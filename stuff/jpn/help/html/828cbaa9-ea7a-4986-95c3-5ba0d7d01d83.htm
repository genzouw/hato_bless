SQLServerStatement のメンバー

次の表は、SQLServerStatement クラスによって公開されるメンバーを示しています。

コンストラクター

なし。

フィールド

なし。

継承されたフィールド

名前

説明

java.sql.Statement

CLOSE_ALL_RESULTS、CLOSE_CURRENT_RESULT、EXECUTE_FAILED、KEEP_CURRENT_RESULT、NO_GENERATED_KEYS、RETURN_GENERATED_KEYS、SUCCESS_NO_INFO

メソッド

名前

説明

addBatch

渡された SQL コマンドを、SQLServerStatement オブジェクトの現在のコマンド一覧に追加します。

cancel

SQLServerStatement オブジェクトによって現在実行されている SQL ステートメントを取り消します。

clearBatch

SQLServerStatement オブジェクトに対する SQL コマンドの現在の一覧を空にします。

clearWarnings

SQLServerStatement オブジェクトについて報告されたすべての警告をクリアします。

close

SQLServerStatement オブジェクトのデータベースと JDBC リソースを、自動的に解放されるまで待たずに直ちに解放します。

execute

渡された SQL ステートメントを実行します。このステートメントは、複数の結果を返す場合があります。

executeBatch

実行するコマンド バッチをデータベースに送信します。すべてのコマンドが正常に実行されている場合は、更新数の配列が返されます。

executeQuery

渡された SQL ステートメントを実行し、1 つの SQLServerResultSet オブジェクトを返します。

executeUpdate

渡された SQL ステートメントを実行します。ステートメントは INSERT、UPDATE、MERGE、DELETE、または SQL DDL ステートメントのような何も返さない SQL ステートメントが可能です。

getConnection

SQLServerStatement オブジェクトを生成した SQLServerConnection オブジェクトを取得します。

getFetchDirection

行をフェッチする方向をデータベース テーブルから取得します。この方向は、SQLServerStatement オブジェクトから生成される結果セットの既定の方向となります。

getFetchSize

結果セットの行数を取得します。この行数は、SQLServerStatement オブジェクトから生成される結果セット オブジェクトの既定のフェッチ サイズとなります。

getGeneratedKeys

SQLServerStatement オブジェクトを実行した結果作成される自動生成キーを取得します。

getMaxFieldSize

SQLServerStatement オブジェクトによって生成される SQLServerResultSet オブジェクトの文字列やバイナリ列の値に対して返すことができる最大バイト数を取得します。

getMaxRows

SQLServerStatement オブジェクトによって生成される SQLServerResultSet オブジェクトが含むことのできる最大行数を取得します。

getMoreResults

SQLServerStatement オブジェクトの次の結果に移動します。

getQueryTimeout

Microsoft JDBC Driver for SQL Server が SQLServerStatement オブジェクトの実行を待つ秒数を取得します。

getResponseBuffering

SQLServerStatement オブジェクトの応答バッファリング モードを取得します。

getResultSet

現在の結果を SQLServerResultSet オブジェクトとして取得します。

getResultSetConcurrency

SQLServerStatement オブジェクトによって生成された SQLServerResultSet オブジェクトの結果セットの同時実行を取得します。

getResultSetHoldability

SQLServerStatement オブジェクトによって生成された SQLServerResultSet オブジェクトの結果セットの保持機能を取得します。

getResultSetType

SQLServerStatement オブジェクトによって生成された SQLServerResultSet オブジェクトの結果セットの種類を取得します。

getUpdateCount

現在の結果を更新数として取得します。

getWarnings

SQLServerStatement オブジェクトでの呼び出しによって報告された最初の警告を取得します。

isClosed

SQLServerStatement オブジェクトが閉じられているかどうかを示します。

isPoolable

ユーザー指定のステートメント プールにステートメントを追加できるかどうかを示す値を返します。

isWrapperFor

ステートメント オブジェクトが指定されたインターフェイスのラッパーであるかどうかを示します。

setCursorName

SQL カーソル名を、渡された文字列に設定します。この文字列は、後に続く実行 (execute) メソッドによって使用されます。

setEscapeProcessing

エスケープの処理モードを設定します。

setFetchDirection

結果セットの行を処理する方向についてのヒントを JDBC ドライバーに示します。

setFetchSize

さらに行が必要な場合にデータベースからフェッチする必要がある行数について、JDBC ドライバーにヒントを示します。

setMaxFieldSize

文字またはバイナリ値を格納する SQLServerResultSet 列の最大バイト数の制限を、渡されたバイト数に設定します。

setMaxRows

任意の SQLServerResultSet オブジェクトが含むことのできる最大行数の制限を、渡された数値に設定します。

setPoolable

ステートメントをプールすること、またはプールしないことを要求します。

setQueryTimeout

SQLServerStatement オブジェクトの実行が完了するのをドライバーが待つ秒数を、渡された秒数に設定します。

setResponseBuffering

SQLServerStatement オブジェクトの応答バッファリング モードを String full または adaptive に設定します。これらのモードの文字列では、大文字と小文字のどちらも使用できます。

unwrap

指定されたインターフェイスを実装するオブジェクトを返します。このメソッドから返されたオブジェクトを使用することで、Microsoft JDBC Driver for SQL Server 固有のメソッドにアクセスできます。
