JDBC Driver の新機能

Microsoft JDBC Driver 4.0 for SQL Server には、いくつかの新機能と機能強化が含まれています。

SQL Azure データベースへの接続に関する情報

SQL Azure データベースへの接続に関するトピックが追加されました。詳細については、「SQL Azure にあるデータベースへの接続」を参照してください。

高可用性、障害回復のサポート

Microsoft JDBC Driver 4.0 for SQL Server は、SQL Server 2012 の AlwaysOn 可用性グループに対する高可用性、障害回復接続をサポートします。詳細については、「高可用性、障害回復のための JDBC Driver のサポート」を参照してください。

Kerberos 統合認証による SQL Server への接続

SQL Server データベースに接続するアプリケーション用に、Microsoft JDBC Driver 4.0 for SQL Server は、タイプ 4 の Kerberos 統合認証を備えています。詳細については、「Kerberos 統合認証による SQL Server への接続」を参照してください。Microsoft JDBC Driver for SQL Server バージョン 4.0 未満では、タイプ 2 の Kerberos 統合認証を利用できます。

拡張イベント ログの診断情報へのアクセス

サーバーの拡張イベント ログにアクセスして、接続エラーに関する情報を得ることができます。詳細については、「拡張イベント ログの診断情報へのアクセス」を参照してください。

スパース列に対するその他のサポート

スパース列が使用されているテーブル内のデータに既にアクセスしているアプリケーションでは、パフォーマンスが向上します。(スパース列の情報を含めて) 列の情報は getColumns メソッド (SQLServerDatabaseMetaData) で取得できます。SQL Server のスパース列の詳細については、「スパース列の使用」を参照してください。

結果セットの列がスパース列セットであるかどうかを示すために、isSparseColumnSet メソッドが SQLServerResultSetMetaData クラスに追加されています。

Xid.getFormatId

Microsoft JDBC Driver 4.0 for SQL Server より前の JDBC Driver では、アプリケーションから形式識別子を受け取っていましたが、その値をデータベース サーバーに渡していませんでした。このとき JDBC Driver では、形式識別子の値として 0 をデータベース サーバーに渡していました。

Microsoft JDBC Driver 4.0 for SQL Server 以降の JDBC Driver では、アプリケーションから形式識別子を受け取ってデータベース サーバーに渡します。このように動作を更新するには、サーバー上の sqljdbc_xa.dll が更新されていることを確認してください。更新されたバージョンの sqljdbc_xa.dll をサーバーにコピーする方法については、「XA トランザクションについて」を参照してください。

Itanium では JDBC Driver 4.0 アプリケーションがサポートされない

Itanium コンピューターでの Microsoft JDBC Driver 4.0 for SQL Server アプリケーションの実行はサポートされません。
